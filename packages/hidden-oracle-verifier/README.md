# Hidden Oracle Verifier Contract

This package contains the Soroban contract that verifies the Groth16 proof generated by the Hidden Oracle frontend.

Source:

- [`src/lib.rs`](./src/lib.rs)
- [`src/generated.rs`](./src/generated.rs)

## Purpose

It receives:

- proof A
- proof B
- proof C
- public signals

and performs the BN254 pairing check on Soroban.

## Generated Constants

`src/generated.rs` is not handwritten. It is generated from the verification key exported by the ZK package.

Source package:

- [`../hidden-oracle-zk`](../hidden-oracle-zk/README.md)

If the circuit or phrase changes, regenerate the artifacts there first, then rebuild/redeploy this contract.

## Build and Test

```bash
cargo build --target wasm32v1-none --release --manifest-path packages/hidden-oracle-verifier/Cargo.toml
```

## Ownership in the App

This verifier contract is called by the worker in the Hidden Oracle flow after the browser has generated the proof locally.

The browser does not submit the on-chain transaction directly.

The worker relays verification and records the resulting Stellar Expert links in the execution trace.
